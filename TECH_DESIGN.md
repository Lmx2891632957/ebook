# 技术设计文档——基于STM32的电子书阅读器

## 1. 项目概述

基于正点原子（ALIENTEK）STM32mini 开发板（MCU为STM32F103RC），在现有 `ebook.c` 例程基础上增强改造，实现一款具有流畅翻页、字号调节、书签管理等功能的电子书阅读器。

---

## 2. 硬件平台
| 组件 | 型号/规格 | 说明 |
|------|-----------|------|
| MCU | STM32F103RC（mini） | 正点原子STM32mini开发板 |
| 显示屏 | 2.8寸TFT LCD屏-ILI9341-ST7789 | 320*240 分辨率，电阻触摸 |
| 存储 | MicroSD 卡（通过 SPI） | 存放 .txt 电子书文件 |
存储资源：主芯片采用自带256K字节FLASH的STM32F103RCT6，并外扩8M字节FLASH

---

## 3. 软件技术栈

| 层次 | 技术组件 | 用途 |
|------|----------|------|
| 实时操作系统 | UCOS-II | 任务调度，管理 GUI 任务与后台任务 |
| HAL 驱动层 | STM32 HAL 库 / 标准外设库 | GPIO、SDIO、DMA、FSMC、PWM 等外设驱动 |
| 文件系统 | FATFS R0.11（ChanFS） | SD 卡文件读写，支持长文件名（LFN），code page 936（中文） |
| 显示驱动 | LCD 驱动（lcd.c / lcd.h） | 像素级绘制、窗口填充、DMA 加速刷屏 |
| 触摸驱动 | 电阻触摸驱动（touch.c / touch.h，XPT2046 控制器，SPI 接口） | 触摸坐标采集、滤波、校准，单点触控 + 滑动手势识别 |
| GUI 框架 | 正点原子 GUI 控件库 | memo（文本显示）、filelistbox（文件浏览）、btn（按钮）、progressbar、window 等控件 |
| 内存管理 | gui_memin_malloc / SRAM heap | 动态分配文本缓冲区和控件内存 |
| 字体系统 | 正点原子字库（SD 卡字库文件） | GBK/GB2312 编码，12/16/24/32 号字体 |

## 4. 功能模块设计

### 4.1 文件浏览与选书（基础功能 #1、#3）

**技术方案**：复用 `filelistbox` 控件。

- 通过 `f_opendir` / `f_readdir` 遍历 SD 卡目录
- 过滤器设为 `FLBOX_FLT_TEXT` 仅显示 .txt 文件
- 支持目录层级进入/退出
- 改进点：增加书籍封面占位图标、文件名截断美化、按首字母排序

### 4.2 文本读取与显示（基础功能 #1）

**技术方案**：基于 FATFS + memo 控件，分段读取策略替代一次性读入。

现有 `ebook.c` 的做法是一次性将整个文件读入内存（`f_txt->obj.objsize + 1` 字节），大文件会内存不足。改进如下：

```
┌──────────────────────────────────────┐
│  SD卡 txt 文件                       │
│  ┌────────────────────────────────┐  │
│  │ 分段缓冲区 (4KB × 2，双缓冲)    │  │
│  │  buffer[0] ← 当前页数据         │  │
│  │  buffer[1] ← 下一页预读数据     │  │
│  └────────────────────────────────┘  │
│            ↓                         │
│  ┌────────────────────────────────┐  │
│  │ 渲染 → LCD 显示区域             │  │
│  └────────────────────────────────┘  │
└──────────────────────────────────────┘
```

- 使用 `f_lseek` + `f_read` 实现分段按需读取
- 双缓冲机制：当前页 + 预读下一页，实现翻页零等待
- GBK 编码解析：中文占 2 字节，ASCII 占 1 字节，需正确处理半字边界
- 换行算法：根据屏幕宽度和当前字号计算每行容纳字数，遇到 `\n` 或 `\r\n` 强制换行

### 4.3 顺滑翻页（基础功能 #2）

**技术方案**：手势识别 + 动画过渡。

```
触摸事件 → 手势判断（滑动方向/距离/速度）
         → 触发翻页动画
              ├── 左右滑动：水平平移旧页+新页（DMA 加速刷屏）
              └── 点击上下区域：无动画直接刷新
```

- 滑动检测：在 `memo_check` 中记录触摸起始/终止坐标，判断 swipe 方向和距离
- 翻页动画：使用 LCD 窗口滚动或帧缓冲平移，约 200-300ms 完成
- 阈值设定：滑动距离 > 屏幕宽度 1/4 触发翻页，否则回弹
- 若硬件支持 DMA2D（如 STM32F7），可利用硬件加速动画

### 4.4 可分档调节字体（基础功能 #4）

**技术方案**：预设字库档位 + memo 重建。

| 档位 | 字号 | 行间距 | 来源 |
|------|------|--------|------|
| 小 | 12px | 2px | 正点原子字库 |
| 中（默认） | 16px | 3px | 正点原子字库 |
| 大 | 24px | 4px | 正点原子字库 |
| 特大 | 32px | 5px | 正点原子字库 |

- 在阅读界面底部或侧边栏增加字号调节按钮（A- / A+）
- 调整字号时，重新计算每页显示行数 → 重建 memo 控件 → 重新排版当前页文本
- 字号设置保存到全局变量，下次打开书籍时沿用

### 4.5 书签功能（基础功能 #5）

**技术方案**：书签数据持久化到 SD 卡小文件。

**数据结构**：

```c
typedef struct {
    char    book_path[256];   // 书籍文件完整路径（用于唯一标识）
    uint32_t bookmark_count;
    struct {
        uint32_t file_offset;  // 书签在文件中的字节偏移
        uint32_t page_num;     // 书签所在页码
        char     description[32]; // 书签备注（可选，取附近文字）
    } bookmarks[10];           // 每本书最多10个书签
} bookmark_db_t;
```

- **添加书签**：阅读界面点击"书签"按钮 → 保存当前 `file_offset` → 写入 `bookmark.dat`
- **查看/跳转**：书签列表界面显示所有书签 → 选择后 `f_lseek` 定位 → 重新排版显示
- **存储位置**：SD 卡根目录 `bookmark.dat`，`f_open` + `f_write` 小文件即可，无需数据库
- 启动时加载书签文件到内存，退出时写回

### 4.6 可拖动进度条（可选功能 #1）

**技术方案**：自定义进度条控件。

- 进度计算：`progress = current_offset / total_file_size × 100%`
- 拖动反馈：触摸进度条滑块 → 实时计算目标 offset → 在旁显示对应页码预览
- 松手后：`f_lseek` 到目标位置 → 重新排版显示
- 每分钟约 5px 精度即可（按 500KB 文件，800px 进度条，每像素约 625 字节）

### 4.7 加载动画（可选功能 #2）

**技术方案**：加载过程中逐帧播放 GIF-like 动画。

- 准备好 4-6 帧简单动画位图（如翻书、旋转圆圈），存储在 SD 卡或内部 Flash
- 在 `f_read` 分段读取循环中，间歇刷新动画帧
- 动画帧之间间隔约 100ms，给用户及时反馈

### 4.8 背景颜色与亮度（可选功能 #3）

**技术方案**：

- **背景色**：修改 `memo` 控件的背景色参数。预设 3-4 种主题（白色/护眼黄/羊皮纸/夜间黑）
- **亮度**：通过定时器 PWM 控制 LCD 背光引脚，占空比 10%-100%
- 在阅读界面侧边栏或设置菜单中提供调节入口

### 4.9 中英文文本读取（可选功能 #4）

**技术方案**：GBK 编码自动兼容 ASCII。

- GBK 编码中，ASCII 字符（0x00-0x7F）占 1 字节，中文（0x81-0xFE 开头）占 2 字节
- 正点原子字库已内置 GBK 字模，`gui_show_string` 可直接渲染混合文本
- 需注意换行时不能在中文两字节之间截断
- 若遇到 UTF-8 编码的英文 txt，可在 PC 端预处理转为 GBK，或集成轻量级 UTF-8→GBK 转换表

## 8. 开发与调试策略

1. **第一阶段**：在正点原子官方综合例程工程基础上编译通过，确认 LCD + SD 卡 + 触摸工作正常
2. **第二阶段**：改造 `ebook.c`，先实现分段读取 + 基础翻页（无动画），跑通文本显示
3. **第三阶段**：叠加字号调节、书签功能
4. **第四阶段**：添加翻页动画、进度条等交互增强
5. **第五阶段**：可选功能（主题、亮度、加载动画）
6. 每一步用 `printf` 通过串口输出调试信息，PC 端用串口助手查看

## 9. 风险与应对

| 风险 | 影响 | 应对措施 |
|------|------|----------|
| 大文件（>500KB）分段读取频繁 `f_lseek` 导致卡顿 | 翻页延迟 | 增大缓冲区到 8KB；利用 DMA 后台预读 |
| GBK 半字截断致乱码 | 显示错误 | 排版时向前回溯 1 字节检查是否 GBK 首字节 |
| 书签文件在写入过程中断电损坏 | 书签丢失 | 写入前先备份旧文件，写入成功后删除备份 |
| 正点原子字库缺少大字号 | 特大字体不可用 | 确认 SD 卡字库文件包含所需字号；若无则取最大可用字号放大 |
| 外部 SRAM 不足 | 多缓冲冲突 | 优先保证当前页缓冲，预读降级为同步读取 |

## 10. 总结

本项目在正点原子 STM32 开发板 + 官方 SDK 框架基础上，对 `ebook.c` 进行增强改造，技术路径清晰、风险可控。所有基础功能和可选功能均可在不改变硬件的前提下通过软件实现。预计代码量约 1500-2500 行（不含框架已有代码），适合作为硬件课程设计项目。
